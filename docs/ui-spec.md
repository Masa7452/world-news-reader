# docs/ui-spec.md
## UI・画面設計書（shadcn/ui 前提）

### デザイン原則
**コンセプト：「世界の話題をゆっくり知ろう☕️」**
- シンプルで見やすいレイアウト（コーヒーブレイクに相応しい落ち着いたデザイン）
- モバイル優先（1カラム）、タブレットで2カラム、デスクトップで3カラム
- 余白を広めに取り、可読性を重視（ゆったりとした読書体験）
- 暖色系のアクセントカラー（コーヒーや木目を連想させる色調）
- ダークモード対応（夜のリラックスタイムにも配慮）
- アクセシビリティ準拠（コントラスト比・フォーカス表示・キーボード操作対応）
- フォント：読みやすさ重視、明朝体とゴシック体のバランス良い組み合わせ

### 画面構成とルーティング
- `/`：最新記事一覧（カードグリッド形式、タグ絞り込み）
- `/news/[slug]`：記事詳細ページ（MDXレンダリング、目次、出典、関連記事）
- `/tags/[tag]`：タグごとの記事一覧
- `/about`：サイトの紹介、利用ポリシー

### レイアウト
- ヘッダー：ロゴ、検索（Cmd+K コマンドパレット）、テーマ切替
- メインコンテンツ：`max-w-[1100px]`、シンプルな余白設計
- フッター：コピーライト、出典ポリシー、SNSリンク

### コンポーネント（shadcn/ui）
- Card（記事カード）：サムネイル／タイトル／タグ／日付を表示
- Badge（タグ表示）：「secondary」スタイル、折り返し対応
- Breadcrumb（記事詳細ヘッダー）
- Separator（章区切り）
- Table（比較記事で利用）
- Alert（注意喚起や免責表記用）
- Dialog / Command（検索UI）
- Skeleton（ローディング）
- Toggle（テーマ切替）

### 一覧画面
- グリッド表示：スマホ1列、タブレット2列、PC3列
- カード内にタイトル（2行省略）、概要、タグを表示
- ページネーション（初期実装）、将来的に無限スクロールへ拡張可

### 記事詳細画面
- タイトル（大きめのフォント）
- 公開日・タグ
- 自動生成の目次（H2/H3から生成）
- 本文（MDX → proseスタイルで整形）
- 画像の場所は `<IMAGE TODO>` として挿入
- 出典セクションを記事末尾に表示（媒体名／URL／日付）
- 関連記事（同タグから5件まで）
- シェアボタン（X/FB/コピーリンク）

### 空状態・エラー処理
- 記事がない場合：人気タグと最新記事を案内
- エラー時：簡潔なメッセージ＋トップへのリンク

### 広告・収益化UI
- ファーストビューには広告を置かない
- 本文H2直下に1枠、本文末尾に1枠、PCサイドに1枠（最大3つ）
- アフィリエイトリンクは「購入はこちら」ボタンとして表示し、出典と区別
